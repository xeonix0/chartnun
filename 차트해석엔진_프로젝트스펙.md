# 차트 해석 엔진 프로젝트 스펙 (가칭: 찰눈)

주단 / 코단 / 비단에서 공용으로 불러 쓰는 **차트 해석·요약 독립 모듈**.
목적: 차트를 직접 보지 않아도, 현재 상태를 구체적인 자연어 문장으로 판단할 수 있게 한다.

---

## 1. 프로젝트 성격

- **독립 프로젝트**로 분리 (주단/코단/비단과 별도 레포/폴더)
- 세 프로젝트의 Scanner 단계에서 **라이브러리 형태로 import**하여 사용
- 입력: 캔들 데이터(OHLCV) — Kiwoom(주단), Bybit(코단/비단) API에서 받은 데이터를 공통 포맷으로 변환하여 전달
- 출력: ①구조화된 데이터(JSON) ②그 데이터를 바탕으로 한 자연어 요약 문장

## 3. 공통 입력 스키마 (schema.py)

```python
@dataclass
class Candle:
    timestamp: int
    open: float
    high: float
    low: float
    close: float
    volume: float

@dataclass
class ChartInput:
    symbol: str            # 종목/코인 코드
    timeframe: str          # "1D", "1H", "15m" 등
    candles: list[Candle]   # 최근 N개 캔들 (충분한 기간, 최소 60봉 권장)
```

각 프로젝트의 adapter가 이 포맷으로 변환해서 core 모듈에 전달.

## 4. 출력 스키마 (구조화 데이터)

```python
@dataclass
class ChartAnalysis:
    symbol: str
    timeframe: str
    trend_direction: str        # "상승" | "하락" | "횡보"
    ma_position: str            # "이평선 위 눌림" | "이평선 아래" | "이평선 근접"
    ma_distance_pct: float      # 현재가와 이평선 간 거리(%)
    resistance_price: float     # 최근 저항선 가격
    resistance_distance_pct: float  # 현재가 대비 저항선까지 거리(%)
    support_price: float        # 최근 지지선 가격
    support_distance_pct: float
    range_state: str            # "박스권 압축중" | "박스권 이탈" | "박스권 아님"
    volume_ratio: float         # 최근 거래량 / 20봉 평균 거래량
    candle_pattern: str | None  # "상승장악형" 등, 없으면 None
    pivot_high_recent: float    # 최근 확정 고점
    pivot_low_recent: float     # 최근 확정 저점
```

## 5. 자연어 요약 예시 (핵심 요구사항: 구체성)

단순히 "상승 추세입니다" 같은 뭉뚱그린 문장이 아니라, **숫자·거리·비율을 포함한 구체적 문장**으로 생성.

### 예시 1 — 눌림목 상황 (주단)

> **[삼성전자, 1D]**
> 현재가는 20일 이평선(71,200원) 대비 **+1.3% 위**에서 눌림 진행 중입니다.
> 최근 확정 고점은 74,500원, 확정 저점은 69,800원이며, 현재가 기준 저항선(74,500원)까지 **+3.9%**, 지지선(69,800원까지) **-2.1%** 남았습니다.
> 최근 거래량은 20봉 평균 대비 **0.8배**로 평소보다 적은 편이며, 뚜렷한 캔들 패턴은 감지되지 않았습니다.
> → 요약: **상승 추세 내 정상 눌림 구간, 거래량 동반 없이 관망 구간으로 판단**

### 예시 2 — 박스권 돌파 임박 (코단)

> **[BTC/USDT, 4H]**
> 최근 18봉 동안 고가-저가 변동폭이 평균 대비 **62% 수준으로 압축**된 박스권 상태입니다.
> 박스 상단은 68,450 USDT, 하단은 66,900 USDT이며, 현재가는 상단까지 **+0.6%** 근접했습니다.
> 최근 거래량은 20봉 평균 대비 **1.4배**로 증가 추세이며, 상단 돌파 시 확인 신호로 볼 수 있습니다.
> → 요약: **박스권 상단 돌파 임박, 거래량 증가 동반 중, 돌파 확인 시 진입 후보**

### 예시 3 — 캔들패턴 + 거래량 결합 (비단)

> **[BTC/USDT, 15m]**
> 직전 봉에서 **상승 장악형 패턴**이 확인되었고, 해당 봉 거래량은 20봉 평균 대비 **1.9배**로 유의미하게 증가했습니다.
> 현재가는 직전 확정 저점(65,120) 대비 **+0.4%** 위에 위치하며, 단기 저항선(65,900)까지 **+1.1%** 남았습니다.
> → 요약: **거래량 동반 반전 신호 발생, 저항선 돌파 여부가 단기 방향성 관건**

## 6. 요약 문장 조합 로직 (summarizer)

1. **추세 판단**: 최근 pivot high/low 시퀀스가 상승/하락/횡보 중 어디에 해당하는지
2. **위치 판단**: 이평선/지지/저항 대비 현재가의 거리(%)를 항상 숫자로 명시
3. **거래량 판단**: 항상 "N배" 형태의 비율로 명시 (정성적 표현 지양)
4. **패턴 판단**: 감지된 캔들패턴이 있으면 명시, 없으면 "특이 패턴 없음"으로 명시적으로 언급
5. **최종 한 줄 요약**: 위 4가지를 조합한 판단 문장 (템플릿 기반, 추후 조건 늘리며 확장)

> 원칙: **모든 문장에 최소 1개 이상의 구체적 수치(가격, %, 배율)를 포함**시켜, "차트를 안 봐도 판단 가능한 수준"을 목표로 함.

## 8. 개발 우선순위

1. `pivot.py` + `ma_relation.py` → 지지/저항 거리 + 눌림목 판단 (가장 기본, 즉시 유용)
2. `summarizer` 기본 템플릿 → 위 두 모듈 결과로 1차 요약 문장 생성

4. `volume_filter.py` + `candle_pattern.py` 추가
5. 주단부터 연동 테스트 → 코단/비단 순차 확장

## 10. 2차 개발 — 비주얼 화면 추가

트레이딩뷰 유료 미가입 상태를 감안해, "비주얼로 확인"하는 경로를 이 프로젝트 안에서 두 가지로 분리해 제공한다.

### 10-1. Python 자체 차트 렌더링 (기본, 트레이딩뷰 불필요)

- **목적**: 트레이딩뷰 없이 로컬/웹에서 바로 시각적으로 확인
- **방식**: `core` 모듈이 계산한 pivot, 추세선, 지지/저항, 박스권 좌표를 그대로 캔들차트 위에 오버레이
- **라이브러리**: `mplfinance`(정적 이미지, 빠름) 또는 `plotly`(인터랙티브, 확대/축소 가능) 중 선택
  - 빠른 확인용 스캐너 결과물 → `mplfinance` (PNG로 저장, Alert 첨부에도 활용 가능)
  - 자세히 들여다보고 싶을 때 → `plotly` (HTML로 저장, 브라우저에서 확대/이동 가능)

**폴더 추가**

```
chart-interpreter/
├── visualizer/
│   ├── static_chart.py     # mplfinance 기반 PNG 생성
│   └── interactive_chart.py # plotly 기반 HTML 생성
```

**출력 예시**

- 캔들차트 위에 표시되는 요소:
  - 확정 고점/저점 마커
  - 추세선 (상승/하락 각각)
  - 박스권 상단/하단 라인
  - 이평선(EMA20 등)
  - 캔들패턴 발생 지점 라벨 (예: "장악형")
- 파일명 규칙: `{symbol}_{timeframe}_{timestamp}.png` 형태로 저장 → Scanner 결과 리스트와 매칭

**연동 예시**

```python
from chart_interpreter.visualizer.static_chart import render_chart

analysis, summary_text = analyze_and_summarize(chart_input)
chart_path = render_chart(chart_input, analysis, save_dir="./output/charts")

# Alert 메시지에 요약 문장 + 차트 이미지 함께 첨부
send_alert(text=summary_text, image_path=chart_path)
```

### 10-2. Pine Script 코드 자동 생성기 (선택적, 트레이딩뷰 붙여넣기용)

- **목적**: 필요할 때만 무료 트레이딩뷰 계정에 붙여넣어 실시간 차트로 재확인하고 싶은 경우
- **방식**: core 모듈의 분석 로직(pivot 탐지, 추세선, 박스권 등)을 Pine Script v6 문법으로 그대로 번역하는 **코드 생성기**를 만든다. 즉 Python으로 직접 계산하는 게 아니라, "Pine Script 소스 코드 문자열"을 만들어내는 모듈.
- 무료 플랜에서도 Pine Script 작성·차트 적용은 가능하므로, 유료 기능(다중 알림, 다중 지표 저장 등) 없이도 단발성 확인용으로 활용 가능

**폴더 추가**

```
chart-interpreter/
├── pinescript_export/
│   ├── templates/            # 기법별 Pine Script 템플릿 (.pine)
│   │   ├── pullback.pine
│   │   ├── range_box.pine
│   │   └── trendline.pine
│   └── generate.py            # 파라미터를 템플릿에 주입해 최종 .pine 코드 생성
```

**동작 방식**

```python
from chart_interpreter.pinescript_export.generate import export_pine_script

pine_code = export_pine_script(
    strategy="trendline",
    params={"leftBars": 5, "rightBars": 5}
)
# pine_code를 .pine 파일로 저장 → 트레이딩뷰 Pine Editor에 복사/붙여넣기
```

- 매번 새로 작성할 필요 없이, 이미 검증된 Python 로직과 동일한 파라미터를 Pine Script에도 그대로 반영해 일관성 유지
- "평소엔 Python 시각화로 확인 → 특정 종목만 트레이딩뷰에서 더 자세히 보고 싶을 때" 선택적으로 사용

### 10-1-1. 비주얼 목업 레퍼런스 (구현 참고용)

실제 구현 전 논의를 통해 확정한 화면 구성 원칙. `static_chart.py` / `interactive_chart.py` 구현 시 아래 기준을 따른다.

**화면에 포함되는 요소 (기법별 on/off 가능해야 함)**

| 요소 | 표시 방식 | 대응 모듈 |
|---|---|---|
| 캔들(양봉/음봉) | 기본 항상 표시 | 원본 OHLCV |
| 이평선(EMA20 등) | 실선, 얇게 | `ma_relation.py` |
| 추세선(저점/고점 연결) | 점선 + 양 끝 점 마커 | `trendline.py` |
| 저항/지지선 | 수평 점선 + 가격 라벨 | `pivot.py` |
| 박스권 압축 구간 | 반투명 박스 영역 | `range_box.py` |
| 돌파/신호 확인 지점 | 원형 마커 강조 | `range_box.py`, `candle_pattern.py` |
| 거래량 바 | 하단 별도 패널, 양봉/음봉 색상 구분 | `volume_filter.py` |
| 범례 | 하단 고정, 사용된 요소만 표시 | 공통 |

**핵심 설계 원칙**

1. **기법 하나당 관련 요소만 켠다.** 모든 요소를 한 화면에 동시에 넣지 않는다.
   - 예: 눌림목 검증 → 캔들 + EMA20 + 가격만
   - 예: 박스권 돌파 검증 → 캔들 + 박스권 + 거래량만
   - 예: 추세선 이탈 검증 → 캔들 + 추세선 + 저항/지지선만
   - `render_chart()` 호출 시 `overlays: list[str]` 파라미터로 표시할 요소를 선택하게 구현 (예: `overlays=["ma", "trendline"]`)
2. **그래프 단독으로 끝내지 않는다.** 반드시 5절의 자연어 요약 문장과 함께 출력해서 "이 선이 왜 여기 있는지"가 텍스트로 연결되도록 한다. (예: 저항선 마커 옆에 "저항선까지 +0.6%" 같은 수치가 요약 문장에도 동일하게 등장해야 함)
3. **색상 의미 고정**: 양봉/상승 관련 = 초록 계열, 음봉/하락 관련 = 빨강 계열, 이평선 = 파랑, 추세선 = 주황, 저항/지지 = 자주색, 박스권 = 보라색으로 프로젝트 전체에서 통일 (주단/코단/비단 모두 동일 색상 규칙 사용 → 나중에 어떤 프로젝트 결과를 봐도 바로 해석 가능)
4. **거래량은 항상 하단 별도 패널**로 분리, 가격 캔들과 색상 규칙(양봉/음봉)을 동일하게 맞춘다.
5. **범례는 실제로 켜진 요소만** 표시 (안 쓴 요소는 범례에서도 숨김 → 화면이 매번 필요한 만큼만 보이게)

### 10-3. 우선순위

1. `visualizer/static_chart.py` (mplfinance PNG) — 가장 빠르게 체감 효과, 트레이딩뷰 의존성 없음
2. Alert 시스템과 이미지 첨부 연동
3. `interactive_chart.py` (plotly HTML) — 필요시 확대해서 보고 싶을 때
4. `pinescript_export` — 여유 있을 때 추가, 필수는 아님
